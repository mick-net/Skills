# Operating Rhythm

Use this to create a weekly solo-founder plan, MRR improvement cadence, or execution checklist.

## Weekly Scorecard

Track only what changes decisions:

- New qualified conversations.
- New trials or signups.
- Activation rate.
- New paid customers.
- MRR or revenue.
- ARPA.
- Churn/cancellations.
- Top 3 churn or non-conversion reasons.
- Support issues repeated more than twice.
- One shipped improvement.
- One public proof asset.

## Weekly Cadence

Monday:
- Review scorecard.
- Confirm current stage/MRR and primary bottleneck.
- Pick one bottleneck.
- Choose one product action and one distribution action.

Tuesday-Wednesday:
- Ship the smallest product improvement.
- Talk to 3-10 users or prospects.

Thursday:
- Turn the shipped improvement or user insight into content.
- Do warm outbound to high-intent leads.

Friday:
- Review results.
- Decide keep, adjust, or cut.
- Write one short lesson for future positioning, pricing, or product scope.

## Daily Founder Actions

Before coding each day, complete at least one customer, revenue, or marketing action unless the current coding task directly fixes activation, payment, retention, support load, or a customer-requested blocker.

Choose 2-4:
- Message 5 warm leads.
- Watch one user try the product.
- Fix one activation blocker.
- Post one product proof update.
- Ask one customer for a story, testimonial, or intro.
- Improve one pricing or landing-page claim.
- Remove one unnecessary step or feature.

## Immediate Action

Every plan should include one action for now, not only the coming week.

Choose one:
- Message a specific buyer.
- Publish or improve one public proof asset.
- Add or test a paid CTA.
- Remove one onboarding blocker.
- Watch one user/session.
- Fix one payment or activation issue.
- Update `company/product.md` with the current stage/MRR if it is missing.

## Monthly Review

Ask:
- Which customer type paid fastest?
- Which channel produced buyers, not just visitors?
- Which feature created the most support?
- Which feature created the most upgrades?
- Which promise converted best?
- What should be removed?
- What should be doubled down?
- Is this still solo-maintainable?

## Solo-Founder Guardrails

- Keep infra boring.
- Avoid custom work unless it teaches repeatable product scope or pays very well.
- Automate repeated support only after seeing the pattern.
- Hire or outsource only for a bottleneck that is already proven.
- Keep a "not now" list to protect focus.
- Prefer one distribution habit over occasional big launches.

## 30-Day MRR Plan Template

Week 1: clarify wedge and buyer.
- Rewrite positioning.
- Talk to 10 prospects/users.
- Cut non-essential roadmap items.

Week 2: improve activation and payment.
- Reduce time-to-value.
- Add or tighten paid boundary.
- Ship one feature that unlocks payment.

Week 3: create proof and distribution.
- Publish demo/customer story.
- Launch or relaunch in one focused channel.
- Start warm outbound queue.

Week 4: convert and retain.
- Follow up every serious lead.
- Fix top churn/non-activation reason.
- Run one pricing or packaging experiment.

End with:
- What moved revenue?
- What created noise?
- What gets repeated next month?

## Brutal Check-In Cadence

Run this weekly when the founder is prone to overbuilding:

1. Inspect the last 7-14 days of commits and docs.
2. Count visible work in three buckets:
   - `Code/product`
   - `Customer/learning`
   - `Marketing/sales`
3. If code/product dominates, force a correction:
   - One customer conversation.
   - One public demo or useful content piece.
   - One pricing/checkout/activation improvement.
4. Append the check-in to `company/coach/solo-founder-growth-coach.md`.

Jon Yongfook-style default:
- Alternate code and marketing weeks, or reserve at least 50% of founder time for growth work until the product has stable demand.
- End each cycle with a newsletter, launch post, demo, tutorial, or customer story.
