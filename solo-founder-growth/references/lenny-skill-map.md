# Lenny Skill Map

Use this when a solo-founder growth task would benefit from Lenny-style depth. Prefer the embedded solo-founder playbooks in `references/lenny-solo-playbooks.md`; installed `lenny-*` skills are optional deeper modules, not the operating system.

If these skills are missing, keep using the embedded playbooks. For source/install pointers, see `references/optional-skill-sources.md`.

## Rule

Read or suggest an external Lenny skill only when the bottleneck is specific and the embedded playbook is not enough. Do not load multiple Lenny skills by default. If the user is overwhelmed, suggest one skill and one next action.

Keep this filter:

- Prefer revenue, customer learning, and shipping over process.
- Translate team/org advice into solo-founder actions.
- Ignore hiring, stakeholder, or department language unless the user already has that scale.
- If a Lenny skill assumes VC-style scale, extract the smallest useful practice.

## Strong Solo-Founder Matches

### First Customers And Sales

Use `lenny-founder-sales` when:
- The user needs first customers, cold/warm outreach, sales calls, discovery, pilots, or early demos.
- The user is still building without talking to buyers.

Solo-founder translation:
- Founder does sales personally.
- Sell before building more.
- Qualify for active pain, not polite interest.
- Close the implementation gap manually if needed.

Also consider:
- `lenny-sales-qualification` once there are enough leads to waste time on bad ones.
- `lenny-enterprise-sales` only for real procurement, security, legal, or multi-stakeholder deals.

### Customer Discovery

Use `lenny-conducting-user-interviews` when:
- The user needs interview scripts, discovery questions, or wants to understand buyer pain.
- The brutal coach asks "who did you talk to?" and there is no answer.

Use `lenny-analyzing-user-feedback` when:
- There are support tickets, interview notes, churn notes, survey responses, reviews, or feedback logs to synthesize.

Solo-founder translation:
- Keep it lightweight: talk to 5-10 real users/prospects, extract repeated language, update positioning or roadmap.

### Scope And MVP

Use `lenny-problem-definition` when:
- The product is hard to explain.
- The user is chasing technology instead of a painful job.
- The request starts with a feature but not a customer problem.

Use `lenny-scoping-cutting` when:
- The user has scope creep, too many features, or needs a weekend-sized MVP.
- The brutal coach finds mostly technical comfort work.

Use `lenny-prioritizing-roadmap` only when:
- There is a real backlog and actual customer/revenue signals to rank.

Solo-founder translation:
- Cut until the next action can produce payment, usage, customer learning, or public proof.

### Positioning And Messaging

Use `lenny-positioning-messaging` when:
- The landing page, one-liner, X launch, or demo copy is unclear.
- The user cannot state who pays and why.

Use `lenny-brand-storytelling` sparingly when:
- There is already some proof and the product needs a memorable narrative.

Solo-founder translation:
- Avoid abstract brand work before traction.
- Write one concrete promise for one buyer and one painful job.

For public website or onboarding audits, start with `references/public-surface-onboarding.md`, then load this skill only if the copy/positioning needs deeper work.

### Marketing And Launch

Use `lenny-content-marketing` when:
- The user wants SEO, blog strategy, technical content, founder-led content, or content-market fit.
- There are docs/features that can become tutorials, free tools, comparison pages, or high-intent search content.

Use `lenny-launch-marketing` when:
- The user is launching a product, feature, Product Hunt post, beta, waitlist, or public announcement.

Use `lenny-community-building` only when:
- The buyer already gathers in a community and the user can contribute there.
- Do not create a community from scratch as a default growth tactic.

Solo-founder translation:
- Jon Yongfook mode: docs, tutorials, demos, free tools, newsletters, and technical posts count as marketing.
- Marc Lou mode: make the launch proof-heavy and memorable.

### Activation, Retention, PMF

Use `lenny-user-onboarding` when:
- Users sign up but do not reach value.
- The first 30-120 seconds are confusing.
- The product needs a faster aha moment.

For landing-page-to-signup or signup-to-aha reviews, start with `references/public-surface-onboarding.md`; load `lenny-user-onboarding` when the activation path itself needs deeper structure.

Use `lenny-retention-engagement` when:
- Users pay but churn, stop using, or fail to form a repeat habit.

Use `lenny-measuring-product-market-fit` when:
- The user asks whether to scale, pivot, or keep iterating.
- There is enough usage/revenue data to judge retention, repeat use, referrals, or willingness to pay.

Solo-founder translation:
- For early products, PMF means paid users repeatedly use it or would complain if it disappeared.

### Pricing And Monetization

Use `lenny-pricing-strategy` when:
- The user is setting prices, changing packaging, deciding free vs paid, or considering freemium.

Solo-founder translation:
- Combine with `references/pricing-monetization.md`.
- One-time pricing only fits low-marginal-cost products.
- Subscription fits hosting, AI usage, teams, sync, support, and ongoing business value.
- Avoid low B2B prices that attract high-support customers.

### PLG To Sales

Use `lenny-product-led-sales` when:
- The product already has self-serve users.
- The user wants to identify product-qualified leads/accounts.
- There are team/enterprise expansion signals.

Solo-founder translation:
- Do not build a sales machine early.
- Start with founder-led outreach to high-intent users based on product behavior.

### Growth Loops

Use `lenny-designing-growth-loops` when:
- There is already activation and retention.
- The user wants referral, viral, content, marketplace, community, or product-led loops.

Solo-founder translation:
- Most pre-PMF founders need direct sales/content first, not clever loops.

### Competitive Analysis

Use `lenny-competitive-analysis` when:
- The user needs positioning against a named competitor.
- Pricing/feature comparison pages are being created.

Solo-founder translation:
- Do not let competitor research replace customer conversations.
- Output should become positioning, a landing page, a comparison page, or sales objection handling.

### AI Products

Use `lenny-ai-evals` when:
- The product's value depends on LLM output quality.
- The user needs rubrics, test cases, regression checks, or quality measurement.

Solo-founder translation:
- Useful when quality affects trust or churn.
- Dangerous if it becomes deep technical work before buyer validation.

### Ideation And Pivots

Use `lenny-startup-ideation` when:
- The user is explicitly exploring new ideas or markets.

Use `lenny-startup-pivoting` when:
- There is persistent weak traction, churn, no willingness to pay, or a product/market mismatch.

Use `lenny-working-backwards` when:
- The user needs to clarify the future customer promise before building.

Solo-founder translation:
- Avoid idea-hopping. Demand evidence before pivoting.

## Suggested Skill Combos

Use at most 2-3 skills together:

- Overbuilding audit: `solo-founder-growth` + `lenny-scoping-cutting` + `lenny-problem-definition`
- First customers: `solo-founder-growth` + `lenny-founder-sales` + `lenny-positioning-messaging`
- Customer discovery: `solo-founder-growth` + `lenny-conducting-user-interviews`
- Landing page: `solo-founder-growth` + `references/public-surface-onboarding.md` + `lenny-positioning-messaging`
- Visual landing execution: `solo-founder-growth` + `references/public-surface-onboarding.md` + `frontend-skill`
- Launch: `solo-founder-growth` + `lenny-launch-marketing` + `lenny-content-marketing`
- SEO/content: `solo-founder-growth` + `lenny-content-marketing`
- Activation: `solo-founder-growth` + `references/public-surface-onboarding.md` + `lenny-user-onboarding`
- Churn: `solo-founder-growth` + `lenny-retention-engagement` + `lenny-analyzing-user-feedback`
- Pricing: `solo-founder-growth` + `lenny-pricing-strategy`
- PLG expansion: `solo-founder-growth` + `lenny-product-led-sales`
- AI quality: `solo-founder-growth` + `lenny-ai-evals`

## How To Suggest A Skill To The User

Use direct phrasing:

`This is now a founder-sales problem. I will use the embedded founder-sales playbook first. If we need deeper outreach/call structure, use $lenny-founder-sales next.`

Or:

`The bottleneck is activation. I will use the embedded onboarding checklist first; if we need more depth, I will read $lenny-user-onboarding and translate it into a solo-founder action plan.`

## Skills To Treat Carefully

- `lenny-enterprise-sales`: only for real enterprise pull.
- `lenny-community-building`: only if a community already exists or the product naturally creates one.
- `lenny-designing-growth-loops`: after activation/retention, not before.
- `lenny-brand-storytelling`: after some evidence, not as a substitute for evidence.
- `lenny-prioritizing-roadmap`: rank evidence-backed work, not speculative ideas.
