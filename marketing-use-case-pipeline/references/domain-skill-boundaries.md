# Domain Skill Boundaries

Reusable domain or workflow skills may describe:

- professional workflow steps;
- evidence categories;
- source-quality rules;
- output structure;
- caveats and human-review requirements.

They should not encode:

- one fictional company;
- exact folder paths or filenames;
- one country or jurisdiction unless the skill is explicitly jurisdiction-specific;
- intentional demo gaps;
- expected final answers;
- customer data or private facts.

Put demo-specific details in the workspace generator, scenario prompt, rubric, or pack README instead.
